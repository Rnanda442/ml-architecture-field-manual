# Presenter scripts

These files are private repository material. They are intentionally outside `app/`, are not imported by the audience application, and must never be displayed by the public site.

Use `complete-presenter-script.ts` as the complete speaker script for all seven flagship cases, the six architecture supplements, and the reusable weight dictionary.

Use `ML_Architecture_Field_Manual_50_Minute_Presentation_Script.txt` for the complete live website presentation. It follows the site from the Home screen through all seven cases, supplements, conclusion, and question buffer, with click, pause, audience-question, slider, and transition cues in parentheses.

## Delivery order

For every flagship case, teach in this order: objective, bottleneck, architecture, lab, then result and limitation.

During the architecture section, move through the diagram one component at a time. Weave each relevant vocabulary and weight explanation into its matching component cue. The `vocabulary` and `weights` collections in the script file are presenter reference material, not separate audience stages. For each selected component, explain what enters, what it does, vocabulary needed there, a simple example, its role in the paper, what leaves, the controlling weight or setting, who controls it, what it changes, and what the audience should inspect next.

Do not open the vocabulary dictionary or weight key during the core walkthrough unless answering a question. Those tools are secondary references.
